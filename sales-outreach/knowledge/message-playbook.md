# Message Playbook — How to Write Messages Anas Will Send

This is the cure for the 2/10 problem. The previous consultant wrote robotic, generic, tech-dump messages. This playbook defines exactly what a good Arkedia message is, with the real bad-vs-good examples from Anas's own history.

---

## THE MESSAGE FORMULA (the order is the law)

1. **Where I found you** — polite greeting + the exact public reason you're reaching out to THIS person. (a keynote, an interview, a signed MoU, a hiring post, a launch). Kills the "who are you / how'd you find me" weirdness.
2. **One real, researched pain** — a single specific problem this person owns, in **plain business English**. Proves you studied them. Not a feature list. Not a stat brag.
3. **Why us (one quiet line)** — confidence that we've solved exactly that. **At most one tech token.**
4. **A bounded, soft ask** — a specific small slot ("15 min?", "20 min next Tue?"). Not "open to a chat?".

Then a 5-word founder identity if not already given: "Anas, founder Arkedia (Al Khobar)."

---

## THE THREE HARD RULES (hardcoded)

1. **Never** open with `FirstName —` (curt dash). Use "Hi [First]," or "Mr/Ms [Last],".
2. **Never** stack more than one tech token. If you wrote "Qdrant", you may NOT also write "Orleans / RAG / .NET 9 / RTL" in the same message.
3. **Never** ask "open to a chat?". Always a specific bounded slot.

## THE BAD vs GOOD CANON (memorize these)

### ❌ BAD (real message Anas rejected — 2/10)
```
[First] — Arkedia (KSA, CR 7040606969). We build the same stack you signed at
LEAP 2025: GPT-4o + Qdrant RAG on .NET 9 / Orleans, AR-EN full RTL, multi-tenant.
Delivered 36+ enterprise platforms for KSA clients from Al Khobar. Would value
15 min on [Company]'s PropTech roadmap.
```
Why it fails:
- Curt `[First] —` opener, no greeting.
- **No "where I found you."** No reason it's aimed at him specifically.
- **Tech dump:** six jargon tokens in one breath. Reads as "I'm pitching," not "I studied your problem."
- **Numbers brag:** "36+ platforms" — every BD person says this.
- **Presumptuous:** "the same stack YOU signed" tells the buyer what they bought.
- **Begging-but-presumptuous ask:** "15 min on [Company]'s PropTech roadmap" = wants free education.

### ✅ GOOD (the version Anas approved — 8/10, ~190 chars)
```
Hi [First], saw your LEAP 2025 keynote. 800+ contractor governance + bilingual
buyer journey at [Company]'s scale is hard - we've shipped that stack for KSA.
Anas, founder Arkedia (Al Khobar). 15 min?
```
Why it works:
- **Polite opener + where-I-found-you:** "Hi [First], saw your LEAP 2025 keynote."
- **One real pain in plain words:** "800+ contractor governance + bilingual buyer journey at [Company]'s scale is hard." Specific, researched, no jargon.
- **Substance not brag:** "we've shipped that stack for KSA." One line.
- **Founder identity in 5 words.** Equal-altitude.
- **Direct bounded ask:** "15 min?"

## TONE CALIBRATION (Anas's stated preferences)
- **Human and formal, not buddy-buddy.** He is not the prospect's friend. "Hi [First], hope you're well" is fine; "Hey man" is not.
- **Founder-to-peer, not vendor-to-buyer.** Bring value/insight; don't ask permission to exist.
- **Plain over clever.** If every vendor says a phrase, drop it. Say the pain like someone who lives it.
- **Short.** Respect their time. LinkedIn connection note ≤ 200 chars. Email tight.
- **No hype, no buzzword soup, no em dashes.**

---

## CHANNEL FORMATS

**LinkedIn connection note (≤200 chars):** the GOOD example above is the template. Formula compressed: greeting + where-found + one pain + one-line why + founder + "15 min?".

**Post-accept LinkedIn DM (longer, after they connect):** same formula, can breathe to ~4–6 sentences. Add ONE relevant proof-of-delivery line ("we built a bilingual real-estate platform across 5 KSA cities") and a concrete next step. Still ≤1 tech token. Still no brag.

**Cold email:** subject = a benefit or a pain in their words (not "Arkedia — software partner"). Body: same formula, 4–7 short sentences, signature block with `anas@arkedia.dev` + `+966 56 385 3092`. Attach ONLY the Arkedia Profile. No NDA/cyber on first touch.

**Follow-up:** never "just following up." Add new value each time (an insight, a relevant case angle, a useful resource). Then a soft check-in. After 2–3 unanswered, a polite break-up that leaves the door open ("should I close the loop? when's better to revisit?").

---

## LABELING (truth discipline)
Every message you hand Anas is **COMPOSED** (drafted by you from sourced facts). When you present it, add a one-line note:
`COMPOSED — angle: [contractor-governance]; leans on facts [where-found source], [pain source].`
So Anas can defend any claim if the prospect asks "where did you read that?".

---

## BEFORE YOU SHOW ANAS — the self-check
Run the rubric in the system prompt. If a draft fails any line, rewrite it. **Never show a failing draft.** Show the recommended message + the one-line angle note. Offer alternatives only if asked.
