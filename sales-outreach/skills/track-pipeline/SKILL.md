# Skill: Track the Outreach Pipeline

Keep every thread alive with a dated next action, and tell Anas the one thing to do next.

## The board
Maintain `pipeline/pipeline.md` (use the pipeline-board template). Stages:
**Target → Connection sent → Connected → Replied → Meeting → Proposal → Won/Lost.**
Per row: prospect, company, stage, channel, last touch (date), **next action (dated)**, angle, risk flag, notes.

## Rules
- **Every open thread has a dated next action.** If one doesn't, create it now.
- **Drop, don't bump forever.** A thread quiet for days with no engagement is usually better replaced by a fresh, better-researched target than bumped a 4th time. Recommend the drop honestly. (The handover notes flagged several dormant threads as drop candidates.)
- **LinkedIn-first discipline:** connection request precedes cold email for senior buyers.
- **Cold first-touch carries only the profile.** NDA/cyber-posture move only after a real reply.

## Each session
1. Update stages and last-touch dates from what happened.
2. Re-time next actions; flag at-risk threads (gone quiet, single-threaded, no reply after N touches).
3. Recommend drops/replacements where bumping is futile.
4. End with **the single most important next action right now** — the one move that most advances real revenue. One person can't do everything; prioritization is the value.

## Current state
Live state (prospects, stages, dated next actions) lives in `pipeline/pipeline.md` and `accounts/`, both kept private and out of the public repo. The handover snapshot was imported there on 2026-06-09; read the board, not this file, for what's current.

## Quality bar
- Board is current; every open row has a dated next action.
- Dead threads called out honestly, not endlessly nursed.
- Anas always leaves with one clear next move.
